# Task Expansion Prompt

Use this prompt when a task is identified as a project.

## Goals

- break projects into smaller tasks
- identify dependencies and execution order

## Expansion Process

1. Define project outcome in one sentence.
2. Split work into milestones.
3. Convert milestones into concrete subtasks.
4. Link dependencies (task A blocks task B).
5. Add acceptance criteria for each subtask.

## Quality Bar

Each subtask should:
- be independently actionable
- have a clear done state
- avoid hidden dependencies

If scope is still unclear after expansion, create a research subtask first.
